# 权限系统说明文档

## 1. 概述

本系统采用基于角色的访问控制（RBAC）模型，通过 `Pid` 字段区分**开发者**和**代理商**两种用户类型，并使用 `Role` 和 `RoleItem` 实现细粒度权限控制。

## 2. 权限模型

### 2.1 用户模型 (Manager)

```go
type Manager struct {
    ID         int       // 用户ID
    Email      string    // 邮箱
    User       string    // 用户名
    Pwd        string    // 密码（明文存储）
    Pid        int       // 关联上级ID，0=开发者，>0=代理商
    PowerId    int       // 关联角色ID
    IsLock     int       // 0正常，1锁定
    IsDel      int       // 0正常，1删除
    // ... 其他字段
}
```

**关键字段说明：**
- `Pid`: 权限分区的核心字段
  - `Pid = 0`: 开发者（最高权限）
  - `Pid > 0`: 代理商（受限权限，值为上级ID）
- `PowerId`: 关联的角色ID，用于代理商的细粒度权限控制

### 2.2 角色模型 (Role)

```go
type Role struct {
    ID          int       // 角色ID
    ManagerId   int       // 归属开发者ID
    Title       string    // 角色名称
    Description string    // 角色功能描述
    CreateTime  time.Time // 创建时间
}
```

角色由开发者创建，用于定义代理商可执行的操作集合。

### 2.3 权限项模型 (RoleItem)

```go
type RoleItem struct {
    ID         int       // 权限项ID
    RoleId     int       // 归属角色ID
    Path       string    // 请求路由（如 /admin/project/createKeys）
    Index      string    // 细粒化请求Key（可选）
    Value      string    // 细粒化请求Value（可选）
    Name       string    // 权限名称（用于错误提示）
    CreateTime time.Time // 创建时间
}
```

**权限匹配规则：**
1. 基础匹配：请求路径与 `Path` 相同
2. 细粒度匹配：若 `Index` 和 `Value` 非空，还需请求参数包含指定值

### 2.4 模型关系图


## 3. 权限校验流程

### 3.1 请求处理流程

```
请求到达
    │
    ▼
┌─────────────────────────────┐
│ BaseController.Prepare()    │
└─────────────────────────────┘
    │
    ├─ 路径是 /admin/user/login? ──是──► 跳过校验
    │
    │ 否
    ▼
┌─────────────────────────────┐
│ 检查 Session Token          │
└─────────────────────────────┘
    │
    ├─ Token 无效? ──是──► 返回 401 "请重新登录"
    │
    │ 否
    ▼
┌─────────────────────────────┐
│ 获取用户信息 (ManagerInfo)   │
└─────────────────────────────┘
    │
    ├─ Pid == 0? ──是──► 开发者，跳过权限检查
    │
    │ 否 (Pid > 0)
    ▼
┌─────────────────────────────┐
│ 获取用户角色权限列表         │
│ (RoleItem by PowerId)       │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 匹配预定义权限路由 RoleList  │
└─────────────────────────────┘
    │
    ├─ 不在权限列表? ──是──► 允许访问
    │
    │ 否
    ▼
┌─────────────────────────────┐
│ 检查用户是否拥有该权限       │
└─────────────────────────────┘
    │
    ├─ 有权限? ──是──► 允许访问
    │
    │ 否
    ▼
返回 400 "暂无权限"
```

### 3.2 Prepare 方法核心逻辑

位置：`controllers/admin/base.go` 第 65-131 行

```go
func (b *BaseController) Prepare() {
    // 1. 登录路由跳过校验
    if strings.ToLower(Uri) != "/admin/user/login" {
        // 2. 验证 Session Token
        id := b.GetSession("token")
        if id == "" || Id <= 0 {
            b.Error(401, "请重新登录")
        }
        
        // 3. 获取用户信息
        b.ManagerInfo = user
        
        // 4. 代理商需要权限检查
        if b.ManagerInfo.Pid > 0 {
            // 获取角色权限列表
            r := models.RoleItem{RoleId: b.ManagerInfo.PowerId}
            list := r.GetUserRole()
            
            // 匹配是否需要权限的路由
            for _, i := range roleList {
                if strings.ToLower(i.Path) == strings.ToLower(Uri) {
                    needCheck = true
                }
            }
            
            // 检查权限
            if needCheck {
                for _, i := range list {
                    if strings.ToLower(i.Path) == strings.ToLower(Uri) {
                        // 细粒度检查
                        if i.Index != "" && i.Value != "" {
                            value := b.GetString(i.Index)
                            if strings.Contains(value, i.Value) {
                                hasPermission = true
                            }
                        } else {
                            hasPermission = true
                        }
                    }
                }
                if !hasPermission {
                    b.Error(400, "暂无权限")
                }
            }
        }
    }
}
```

## 5. 当前问题与优化建议

### 5.1 问题列表

| 序号 | 问题 | 严重程度 | 位置 |
|------|------|----------|------|
| 1 | 密码明文存储 | 高 | `models/manager.go` Login 方法 |
| 2 | 权限标签过于简单 | 中 | `models/manager.go` Permission 定义 |
| 3 | RoleList 硬编码 | 中 | `models/roleItem.go` RoleList 变量 |
| 4 | 无权限继承机制 | 中 | 权限模型设计 |
| 5 | 缺少操作日志 | 低 | 权限校验流程 |
| 6 | 错误信息暴露路径 | 低 | `base.go` 错误返回 |

### 5.2 优化建议

#### 5.2.1 密码加密存储（高优先级）

```go
import "golang.org/x/crypto/bcrypt"

// 注册/修改密码时
hashedPassword, _ := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
user.Pwd = string(hashedPassword)

// 登录验证时
err := bcrypt.CompareHashAndPassword([]byte(user.Pwd), []byte(password))
if err != nil {
    return "密码错误", nil
}
```

#### 5.2.2 增强权限标签（中优先级）

```go
type Permission struct {
    Label    string   `json:"label"`
    Value    string   `json:"value"`
    Actions  []string `json:"actions"` // 具体操作列表
}

// 示例
var AgentPermission = []Permission{
    {
        Label:   "激活码管理员",
        Value:   "key_manager",
        Actions: []string{"create", "lock", "delete", "batch"},
    },
}
```

#### 5.2.3 权限配置外部化（中优先级）

将 `RoleList` 移至数据库或配置文件：

```yaml
# config/permissions.yml
permissions:
  - name: 激活码管理
    children:
      - name: 创建激活码
        path: /admin/project/createKeys
      - name: 锁定/解锁激活码
        path: /admin/project/lockKey
```

#### 5.2.4 权限继承机制（中优先级）

```go
type Role struct {
    ID          int
    ParentId    int    // 父角色ID，实现继承
    Title       string
    Description string
}

// 获取权限时合并父角色权限
func (r *Role) GetAllPermissions() []RoleItem {
    permissions := r.GetRoleItems()
    if r.ParentId > 0 {
        parent := Role{ID: r.ParentId}
        permissions = append(permissions, parent.GetAllPermissions()...)
    }
    return permissions
}
```

#### 5.2.5 操作审计日志（低优先级）

```go
type AuditLog struct {
    ID         int
    UserId     int
    Action     string
    Resource   string
    Result     string // success/denied
    IP         string
    CreateTime time.Time
}

// 在权限校验时记录
func (b *BaseController) logAccess(result string) {
    log := AuditLog{
        UserId:   b.ManagerId,
        Action:   b.Ctx.Request.Method,
        Resource: b.Ctx.Request.RequestURI,
        Result:   result,
        IP:       b.Ctx.Request.RemoteAddr,
    }
    // 保存日志
}
```
