## Context

会员管理系统提供了一套完整的API接口，用于第三方软件进行用户验证、权限管理、充值扣点等功能。当前已有Apipost格式的接口文档，但缺少面向开发者的Markdown格式对接文档。

**背景信息：**
- 项目名称：会员管理系统
- 接口数量：18个
- 主要功能：用户认证、权限验证、充值扣点、心跳检测
- 通信协议：HTTP POST
- 数据格式：JSON

## Goals / Non-Goals

**Goals:**
- 产出清晰的API对接文档，便于第三方开发者快速接入
- 包含完整的接口说明、请求参数、响应格式、错误码
- 提供签名算法说明和安全机制介绍
- 包含请求示例和响应示例

**Non-Goals:**
- 不修改任何代码实现
- 不涉及接口功能变更
- 不涉及性能优化

## Decisions

### 1. 文档结构设计

采用分层结构组织文档：
- **概述部分**：系统介绍、认证机制、签名算法
- **公共参数**：所有接口通用的请求头、响应格式
- **接口详情**：按功能模块分组，每个接口包含完整说明
- **错误码对照表**：常见错误码及处理建议

**理由**：分层结构便于开发者按需查阅，降低学习成本。

### 2. 接口分组方式

按功能模块分组：
- **基础接口**：时间戳、获取权限、获取软件信息、获取会员标签、获取远程变量
- **验证码接口**：验证码接口、注册验证码邮件、找回账号验证码邮件
- **用户管理接口**：用户注册、用户登录、用户解绑、用户下线、用户心跳
- **账户操作接口**：用户充值、账号扣点、账号拉黑、查询用户在线、找回账号

**理由**：功能分组便于开发者快速定位所需接口。

### 3. 签名算法说明

文档需详细说明签名生成流程：
1. 获取时间戳
2. 组合签名字符串：`appkey + secretkey + version + timestamp + mac`
3. MD5加密生成签名
4. 请求携带签名参数

**理由**：签名是安全认证的核心，必须清晰说明。

## Risks / Trade-offs

**风险：**
- [文档与实际接口不一致] → 定期同步更新，标注文档版本和更新日期
- [签名算法变更未及时通知] → 文档中注明版本兼容性说明

**权衡：**
- 文档详细程度 vs 可读性：选择适度详细，关键信息突出显示
