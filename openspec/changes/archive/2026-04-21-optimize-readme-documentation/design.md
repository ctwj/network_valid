## Context

当前 README.md 内容简单，仅包含部署说明。需要扩展为完整的项目文档，包括功能介绍、架构分析、优劣势评估。

## Goals / Non-Goals

**Goals:**
- 提供完整的系统功能概览
- 分析系统架构和技术选型
- 评估系统优势和劣势
- 提出后续优化方向

**Non-Goals:**
- 不涉及代码修改
- 不创建独立的 API 文档（仅概览）

## Decisions

### 文档结构设计

采用以下结构组织 README.md：

1. **项目简介** - 一句话描述系统定位
2. **功能模块** - 按模块列出功能点
3. **技术架构** - 技术栈和架构特点
4. **快速开始** - 部署和配置说明
5. **系统优势** - 架构和功能亮点
6. **改进方向** - 劣势和优化建议
7. **API 概览** - 开放接口列表

### 内容来源

- 功能模块：从 controllers 和 models 代码分析得出
- 架构特点：从现有代码结构总结
- 优劣势：基于代码审查和技术评估

## Risks / Trade-offs

- **文档维护成本** → 保持简洁，聚焦核心功能
- **内容过时风险** → 在代码变更时同步更新文档
