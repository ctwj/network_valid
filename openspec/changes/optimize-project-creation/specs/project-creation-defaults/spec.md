## ADDED Requirements

### Requirement: 项目创建时自动创建默认登录规则
系统在创建新项目时，SHALL 自动创建一条默认登录规则并绑定到该项目。

#### Scenario: 创建项目时自动创建默认登录规则
- **WHEN** 用户创建新项目
- **THEN** 系统自动创建登录规则，配置为：绑定模式=普通登录(mode=1)，注册模式=普通注册(reg_mode=1)，解绑模式=任意解绑(unbind_mode=3)
- **AND** 该规则自动绑定到新创建的项目

### Requirement: 项目创建时自动创建初始版本号
系统在创建新项目时，SHALL 自动创建一条初始版本号记录。

#### Scenario: 创建项目时自动创建初始版本号
- **WHEN** 用户创建新项目
- **THEN** 系统自动创建版本号记录，版本号为 1.00，状态为启用

### Requirement: 套餐方案支持动态月费定价
系统 SHALL 支持基于单月费用动态计算套餐方案金额。

#### Scenario: 用户修改单月费用
- **WHEN** 用户在创建项目时修改单月费用（默认 30 元）
- **THEN** 所有套餐方案的金额基于新的月费重新计算
- **AND** 保持各方案间的价格比例关系

### Requirement: 运营模式配置提示
系统 SHALL 在创建项目 UI 中提供运营模式对 SDK 影响的说明。

#### Scenario: 用户选择运营模式
- **WHEN** 用户查看或选择运营模式
- **THEN** 系统显示该模式对 SDK 调用的影响说明

### Requirement: 加密模式配置提示
系统 SHALL 在创建项目 UI 中提供加密模式对 SDK 影响的说明。

#### Scenario: 用户选择加密模式
- **WHEN** 用户查看或选择加密模式
- **THEN** 系统显示该模式对 SDK 配置的影响说明

### Requirement: 签名算法配置提示
系统 SHALL 在创建项目 UI 中提供签名算法对 SDK 影响的说明。

#### Scenario: 用户选择签名算法
- **WHEN** 用户查看或选择签名算法
- **THEN** 系统显示 SDK 需使用的对应签名算法
