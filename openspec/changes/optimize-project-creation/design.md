## Context

当前项目创建流程分散在多个步骤中：
1. 创建项目基础信息
2. 手动创建登录规则并绑定
3. 手动创建版本号
4. 套餐方案金额固定，无法灵活调整

用户期望"一步创建即可对接 SDK"，需要后端在创建项目时自动完成关联配置。

**约束条件**：
- 后端使用 Beego ORM
- 前端使用 Vue 3 + Naive UI
- SDK 已有默认签名算法为 MD5，需保持兼容

## Goals / Non-Goals

**Goals:**
- 项目创建时自动创建默认登录规则（普通登录、普通注册、任意解绑）
- 项目创建时自动创建初始版本号（1.00）
- 套餐方案支持动态月费基准，修改后套餐金额同步更新
- UI 提供运营模式/加密模式/签名算法的友好提示

**Non-Goals:**
- 不改变现有登录规则的数据结构
- 不改变现有版本号的数据结构
- 不影响已创建项目的配置

## Decisions

### 1. 默认登录规则配置
**决定**：在 `Project.Add` 方法中，创建项目后立即创建默认登录规则并绑定

**字段值**：
- `Mode = 1`（普通登录）
- `RegMode = 1`（普通注册）
- `UnbindMode = 3`（任意解绑）
- `Title = "默认规则"`

**理由**：这是最宽松的配置，适合快速对接 SDK 测试，用户后续可按需调整。

### 2. 默认版本号配置
**决定**：在 `Project.Add` 方法中，创建项目后立即创建初始版本号

**字段值**：
- `Version = 1.00`
- `IsActive = 0`（启用）
- `IsMustUpdate = 0`（不强制更新）

**理由**：SDK 对接需要版本号验证，默认 1.00 是常见初始版本。

### 3. 套餐方案动态定价
**决定**：前端新增"单月费用"输入框，套餐金额基于此计算

**计算规则**（以月费 M 为基准）：
- 入门引导方案：月卡 M，季卡 M×2.67（约省 10%）
- 标准推荐方案：月卡 M，季卡 M×2.63（约省 12%），年卡 M×8.14（约省 32%）
- 高级专业方案：月卡 M×3.3，季卡 M×8.3，年卡 M×23.3，永久 M×66.6

**理由**：保持现有价格比例关系，用户只需调整一个参数即可整体定价。

### 4. UI 提示优化
**决定**：在现有 Alert 组件基础上，增加对 SDK 影响的说明

**新增提示**：
- 运营模式：说明收费/免费对 SDK 调用的影响
- 加密模式：说明 AES 加密需要 SDK 配置密钥
- 签名算法：说明 SDK 需使用对应签名算法

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 默认登录规则可能不符合所有用户需求 | 提示用户可在项目详情中修改规则 |
| 套餐金额自动计算可能导致非整数价格 | 前端显示时保留两位小数，后端存储支持浮点 |
| 自动创建的版本号可能与现有版本冲突 | 仅在项目创建时创建，不处理后续版本 |
