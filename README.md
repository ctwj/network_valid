# 软中心 (Soft-Center)

一个**软件授权验证管理系统**，用于管理软件产品的激活码、会员、代理等业务，支持多项目、多代理的 SaaS 模式运营。

## 功能模块

### 项目管理

- 创建/编辑/删除项目
- 支持两种项目类型：
  - **单码模式**：激活码即账号，无需注册
  - **账号模式**：用户注册账号后使用激活码充值
- 项目状态管理：收费运营 / 停止运营 / 免费运营
- 自动生成 RSA 密钥对（公钥/私钥）
- AES 加密密钥管理（KeyA/KeyB）
- API 签名算法配置（MD5/SHA1~SHA512）

### 激活码管理

- 激活码类型管理：名称、价格、天数、点数、前缀、标签
- 激活码生成：
  - 支持格式：纯数字、大写字母数字、小写字母数字、随机组合
  - 自定义长度和前缀
  - 批量生成
- 激活码状态管理：锁定/解锁/删除
- 激活码查询：按项目、卡类、使用状态、会员筛选
- 批量操作：批量锁定、解锁、删除

### 会员管理

- 会员注册/登录（支持邮箱或用户名）
- 会员信息管理：密码、安全密码、天数、点数、MAC地址
- 会员状态控制：锁定/解锁
- 解绑功能：
  - 支持解绑次数限制
  - 解绑时间间隔限制
  - 解绑扣费（天数/点数）
- 在线会员管理：查看在线列表、强制下线

### 代理系统

- 多级代理体系：树形结构管理
- 代理账号管理：创建/编辑/删除
- 代理余额充值与管理
- 代理授权管理：
  - 授权特定激活码类型
  - 设置代理专属价格
- 代理权限角色配置

### 版本管理

- 版本号创建与管理
- 版本状态：启用/禁用
- 强制更新控制
- 热更新地址配置
- 版本更新公告

### 登录规则

- 三种登录模式：
  1. **绑定登录**：设备MAC绑定
  2. **普通登录**：无绑定
  3. **点数登录**：按点数控制多开
- 多开控制：单机多开数量限制、全局多开数量限制
- 解绑规则：自动解绑/手动解绑、次数限制、时间间隔限制、扣费规则
- 注册控制：是否需要激活码注册、邮箱注册开关

## 技术架构

### 技术栈

| 层次 | 技术选型 |
|------|---------|
| 前端框架 | Vue 3 + TypeScript |
| UI 框架 | Naive UI + TailwindCSS |
| 构建工具 | Vite |
| 后端框架 | Go + Beego v2 |
| 数据库 | SQLite / MySQL（可配置） |
| 缓存 | 文件缓存 / Redis |
| 部署方式 | 单文件嵌入静态资源 |

### 架构特点

- **前后端一体**：后端嵌入静态文件，支持 SPA 路由回退
- **多数据库支持**：SQLite 开箱即用，MySQL 支持大规模部署
- **加密传输**：支持 AES 加密 API，保护数据安全
- **多级代理**：树形结构支持无限层级代理体系
- **缓存优化**：项目信息、版本信息、会员信息多级缓存

## 快速开始

### 1. 配置文件设置

首次运行前，需要创建配置文件：

```bash
cd backend
cp config.conf.example config.conf
```

编辑 `config.conf` 文件，配置数据库和缓存信息：

```ini
[app]
cache    = file    # 缓存类型: file 或 redis
key      = your-secret-key

[sql]
type     = sqlite  # 数据库类型: sqlite 或 mysql
ip       = 127.0.0.1
port     = 3306
user     = your_user
pwd      = your_password
db       = your_database
rebuild  = false
```

### 2. 本地开发

#### 前端开发

```bash
cd frontend
pnpm install
pnpm dev
```

#### 后端开发

```bash
cd backend
go mod tidy
go run main.go
```

### 3. 构建生产版本

#### 手动构建

```bash
# 1. 构建前端
cd frontend
pnpm install
pnpm build

# 2. 复制前端构建产物到后端
cp -r dist ../backend/static

# 3. 构建后端
cd ../backend
go build -o soft-center .
```

#### GitHub Actions 自动构建

1. 进入 GitHub 仓库的 Actions 页面
2. 选择 "Build and Release" 工作流
3. 点击 "Run workflow"
4. 输入版本号（如 `v1.0.0`）
5. 点击 "Run workflow"

构建完成后，会在 Releases 页面生成对应版本的二进制文件：
- `soft-center-linux-amd64` - Linux 版本
- `soft-center-windows-amd64.exe` - Windows 版本

## 部署

### 单文件部署

构建后的二进制文件已包含前端静态资源，只需：

1. 下载对应平台的二进制文件
2. 在同目录创建 `config.conf` 配置文件
3. 运行程序

```bash
./soft-center
```

默认监听端口为 9960，访问 `http://localhost:9960` 即可。

## 系统优势

### 架构优势

- **单一二进制部署**：前端静态资源嵌入 Go 二进制，无需 Nginx 配置
- **跨平台支持**：一次构建，Linux/Windows 通用
- **低依赖启动**：SQLite 模式下无需外部数据库服务

### 功能优势

- **多项目隔离**：支持多项目独立配置和运营
- **灵活的登录模式**：绑定登录、普通登录、点数登录三种模式
- **多级代理体系**：支持无限层级代理，灵活的价格体系
- **完善的加密机制**：AES 加密传输、RSA 签名验证

### 运维优势

- **GitHub Actions CI/CD**：一键构建发布
- **配置模板化**：敏感配置不入库，便于安全部署

## API 概览

### 开放接口 (`/api`)

| 接口 | 说明 |
|------|------|
| `soft.init` | 软件初始化，获取项目信息、版本、公告 |
| `user.login` | 会员登录（单码/账号模式） |
| `user.register` | 会员注册（账号模式） |
| `user.heartbeat` | 心跳检测，维持在线状态 |
| `user.recharge` | 激活码充值续费 |
| `user.unbind` | 解绑设备 |

### 管理接口 (`/admin`)

| 模块 | 接口前缀 | 说明 |
|------|---------|------|
| 用户 | `/admin/user/*` | 登录、登出、信息获取 |
| 项目 | `/admin/project/*` | 项目 CRUD、激活码管理 |
| 会员 | `/admin/project/*` | 会员管理、在线管理 |
| 代理 | `/admin/project/*` | 代理管理、授权管理 |

### 安全机制

- **API 签名验证**：请求参数签名校验
- **AES 加密传输**：敏感数据加密
- **时间戳防重放**：请求时效性验证
- **Token 认证**：管理端 Token 验证
