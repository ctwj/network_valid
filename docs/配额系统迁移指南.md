# 配额系统迁移指南

本文档说明现有项目如何启用配额系统。

## 概述

配额系统是可选功能，现有项目可以：
1. 不启用，保持原有逻辑
2. 启用，为现有用户配置配额

## 数据库迁移

### 1. 执行数据库变更

```sql
-- 1. cards 表新增字段
ALTER TABLE cards ADD COLUMN quota_rules TEXT NULL COMMENT '配额规则JSON';
ALTER TABLE cards ADD COLUMN is_free_tier TINYINT DEFAULT 0 COMMENT '是否为免费套餐';
ALTER TABLE cards ADD COLUMN priority INT DEFAULT 0 COMMENT '套餐优先级';

-- 2. 创建 user_quotas 表
CREATE TABLE user_quotas (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    member_id INT NOT NULL,
    quota_key VARCHAR(64) NOT NULL,
    limit_value BIGINT DEFAULT 0,
    used_value BIGINT DEFAULT 0,
    period VARCHAR(16) NOT NULL,
    reset_day INT DEFAULT 1,
    reset_time DATETIME NOT NULL,
    expire_time DATETIME NULL,
    plan_id INT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_member_id (member_id),
    INDEX idx_plan_id (plan_id),
    UNIQUE KEY uk_member_quota_plan (member_id, quota_key, plan_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 3. 创建 member_plans 表
CREATE TABLE member_plans (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    member_id INT NOT NULL,
    plan_id INT NOT NULL,
    status VARCHAR(16) DEFAULT 'active',
    expire_time DATETIME NULL,
    activated_at DATETIME NOT NULL,
    expired_at DATETIME NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_member_id (member_id),
    INDEX idx_plan_id (plan_id),
    INDEX idx_status (status),
    INDEX idx_expire_time (expire_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 4. 创建 quota_usage_logs 表（可选）
CREATE TABLE quota_usage_logs (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    member_id INT NOT NULL,
    quota_key VARCHAR(64) NOT NULL,
    amount BIGINT NOT NULL,
    plan_id INT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_member_id (member_id),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 5. 添加系统配置项
INSERT INTO system_config (config_key, config_value, description)
VALUES ('PlanPresets', '[]', '预设套餐方案JSON');
```

## 现有用户迁移

### 方案一：不迁移，新用户启用

现有用户不启用配额系统，新注册用户自动获得免费套餐。

**优点**：无迁移成本，平滑过渡
**缺点**：现有用户和新用户体验不一致

### 方案二：批量迁移，为现有用户发放免费套餐

```go
// 迁移脚本
func MigrateExistingUsers() {
    o := orm.NewOrm()

    // 获取所有现有用户
    var members []models.Members
    o.Raw("SELECT id FROM members").QueryRows(&members)

    for _, member := range members {
        // 检查是否已有套餐
        exists := o.QueryTable("member_plans").
            Filter("member_id", member.Id).
            Exist()

        if !exists {
            // 获取项目的免费套餐
            var freeTier models.Cards
            err := o.QueryTable("cards").
                Filter("project_id", member.ProjectId).
                Filter("is_free_tier", 1).
                One(&freeTier)

            if err == nil {
                // 发放免费套餐
                planService.GrantFreeTier(member.Id, freeTier.Id)
            }
        }
    }
}
```

### 方案三：根据现有数据迁移

如果现有用户已有天数/点数，可以根据情况迁移：

```go
func MigrateWithExistingData() {
    o := orm.NewOrm()

    var members []models.Members
    o.Raw("SELECT id, endtime FROM members WHERE endtime > NOW()").QueryRows(&members)

    for _, member := range members {
        // 计算剩余天数
        days := int(member.Endtime.Sub(time.Now()).Hours() / 24)

        if days > 0 {
            // 用户还有剩余时间，创建对应的套餐记录
            // ...
        } else {
            // 用户已过期，发放免费套餐
            planService.GrantFreeTier(member.Id)
        }
    }
}
```

## 配置配额规则

### 为现有套餐类型添加配额规则

```sql
-- 更新现有套餐类型，添加配额规则
UPDATE cards
SET quota_rules = '{"quotas": [{"key": "download", "name": "下载次数", "limit": 20, "period": "daily", "unit": "count"}]}',
    priority = 10
WHERE title = '月卡' AND is_free_tier = 0;

UPDATE cards
SET quota_rules = '{"quotas": [{"key": "download", "name": "下载次数", "limit": 50, "period": "daily", "unit": "count"}, {"key": "traffic", "name": "流量", "limit": 1073741824, "period": "monthly", "reset_day": 1, "unit": "bytes"}]}',
```

## 启用定时任务

配额系统需要两个定时任务：

### 1. 配额重置任务

```go
// 每小时执行一次
func QuotaResetTask() {
    quotaService.ResetExpiredQuotas()
}
```

### 2. 套餐到期检查任务

```go
// 每小时执行一次
func PlanExpireCheckTask() {
    planService.CheckExpiredPlans()
}
```

### 配置定时任务

在 `main.go` 或任务调度器中添加：

```go
// 使用 cron 库
c := cron.New()

// 每小时执行配额重置
c.AddFunc("0 * * * *", func() {
    quotaService.ResetExpiredQuotas()
})

// 每小时执行套餐到期检查
c.AddFunc("0 * * * *", func() {
    planService.CheckExpiredPlans()
})

c.Start()
```

## 客户端更新

### SDK 更新

更新 SDK 以支持配额接口：

```go
// 查询配额
quotas, _ := client.CheckQuota(nil)

// 扣减配额
result, _ := client.DeductQuota("download", 1)

// 获取当前套餐
plan, _ := client.GetCurrentPlan()
```

### 兼容性

配额接口是新增接口，不影响现有接口：
- 现有登录、充值、心跳等接口保持不变
- 配额接口为可选调用
- 不调用配额接口不影响基本功能

## 回滚方案

如需回滚配额系统：

```sql
-- 1. 删除新增表
DROP TABLE IF EXISTS quota_usage_logs;
DROP TABLE IF EXISTS member_plans;
DROP TABLE IF EXISTS user_quotas;

-- 2. 删除新增字段
ALTER TABLE cards DROP COLUMN quota_rules;
ALTER TABLE cards DROP COLUMN is_free_tier;
ALTER TABLE cards DROP COLUMN priority;

-- 3. 删除系统配置
DELETE FROM system_config WHERE config_key = 'PlanPresets';
```

## 注意事项

1. **备份数据**：迁移前务必备份数据库
2. **测试环境**：先在测试环境验证迁移脚本
3. **分批迁移**：大量用户时分批迁移，避免锁表
4. **监控告警**：迁移后监控配额相关接口和定时任务
5. **用户通知**：如有必要，通知用户新的配额规则
