# 客户端配额交互流程

本文档描述客户端如何使用配额系统进行使用限制检查和扣减。

## 概述

配额系统允许对用户使用进行多维度限制，客户端需要在关键操作前检查配额，操作后扣减配额。

## 交互流程

### 1. 登录后获取配额信息

用户登录成功后，客户端应查询当前配额状态，用于 UI 展示。

```go
// 登录
userInfo, _ := client.Login(loginReq)

// 查询所有配额
quotas, _ := client.CheckQuota(nil)

// 展示给用户
for _, q := range quotas {
    fmt.Printf("%s: %d/%d\n", q.Name, q.Used, q.Limit)
}
```

### 2. 操作前检查配额

在执行受限操作前，检查配额是否充足。

```go
// 检查下载配额
quotas, _ := client.CheckQuota([]string{"download"})

if len(quotas) == 0 {
    fmt.Println("无配额信息")
    return
}

quota := quotas[0]
if !quota.Unlimited && quota.Remaining <= 0 {
    fmt.Println("下载次数已用完")
    // 提示用户升级套餐或等待重置
    return
}

// 继续执行下载操作
doDownload()
```

### 3. 操作后扣减配额

操作完成后，扣减相应配额。

```go
// 执行下载
err := doDownload()
if err != nil {
    return // 失败不扣减
}

// 扣减配额
result, err := client.DeductQuota("download", 1)
if err != nil {
    // 网络错误，可能需要重试或本地记录
    logError(err)
}

if !result.Success {
    fmt.Printf("扣减失败: %s\n", result.Message)
}
```

### 4. 批量操作

涉及多个配额的操作，使用批量扣减保证原子性。

```go
// 检查所有相关配额
quotas, _ := client.CheckQuota([]string{"download", "traffic"})

// 验证配额充足
for _, q := range quotas {
    if !q.Unlimited && q.Remaining <= 0 {
        fmt.Printf("%s 不足\n", q.Name)
        return
    }
}

// 执行操作
err := doOperation()
if err != nil {
    return
}

// 批量扣减
results, _ := client.DeductQuotaBatch([]sdk.QuotaDeductItem{
    {Key: "download", Amount: 1},
    {Key: "traffic", Amount: fileSize},
})

// 检查结果
for _, r := range results {
    if !r.Success {
        fmt.Printf("%s 扣减失败: %s\n", r.Key, r.Message)
    }
}
```

## 典型场景

### 场景一：文件下载

```
用户点击下载
    │
    ▼
检查 download 配额 ──不足──▶ 提示"下载次数已用完"
    │
    充足
    │
    ▼
执行下载
    │
    ├─ 成功 ──▶ 扣减 download 配额
    │
    └─ 失败 ──▶ 不扣减，提示错误
```

### 场景二：API 调用

```
客户端发起 API 请求
    │
    ▼
检查 api_call 配额 ──不足──▶ 返回错误"API调用次数已用完"
    │
    充足
    │
    ▼
执行 API 调用
    │
    ├─ 成功 ──▶ 扣减 api_call 配额
    │
    └─ 失败 ──▶ 不扣减
```

### 场景三：流量使用

```
用户上传/下载文件
    │
    ▼
检查 traffic 配额 ──不足──▶ 提示"流量不足"
    │
    充足
    │
    ▼
执行传输（记录实际传输字节数）
    │
    ▼
扣减 traffic 配额（按实际字节数）
```

## 错误处理

### 配额不足

```go
result, err := client.DeductQuota("download", 1)
if err == nil && !result.Success {
    // 配额不足或套餐过期
    fmt.Println(result.Message) // "配额不足" 或 "套餐已过期"

    // 引导用户升级
    suggestUpgrade()
}
```

### 网络错误

```go
result, err := client.DeductQuota("download", 1)
if err != nil {
    // 网络错误，可能需要重试
    // 建议：本地记录待扣减，下次同步
    logPendingDeduction("download", 1)
}
```

### 套餐过期

```go
plan, _ := client.GetCurrentPlan()
if plan.Status == "expired" {
    // 付费套餐过期，已降级为免费套餐
    fmt.Println("套餐已过期，已降级为免费套餐")

    // 重新查询配额（免费套餐的配额）
    quotas, _ := client.CheckQuota(nil)
}
```

## 最佳实践

1. **先查后扣**：操作前检查，操作后扣减
2. **失败不扣**：操作失败时不扣减配额
3. **批量原子**：多配额操作使用批量扣减
4. **本地缓存**：缓存配额状态，减少 API 调用
5. **错误重试**：网络错误时记录待扣减，稍后重试
6. **友好提示**：配额不足时引导用户升级或等待重置

## 配额重置时机

| 周期 | 重置时机 |
|------|----------|
| daily | 每日 00:00 |
| weekly | 每周一 00:00（可配置为其他周几） |
| monthly | 每月 1 日 00:00（可配置为其他日期） |

客户端可通过 `ResetTime` 字段获知下次重置时间，用于 UI 倒计时展示。
