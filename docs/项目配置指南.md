## 1. 项目基础配置

### 1.1 项目属性说明

| 属性 | 说明 | 可选值 |
|------|------|--------|
| **登录模式** | 控制用户认证方式 | 单码 / 用户 |
| **运营模式** | 控制项目运营状态 | 收费运营 / 停止运营 / 免费运营 |
| **加密模式** | API 通信加密方式 | 开放API / AES |
| **签名算法** | 请求签名算法 | MD5 / SHA1 / SHA224 / SHA256 / SHA384 / SHA512 |

### 1.2 登录模式详解

#### 单码模式 (Type=0)
- 用户直接使用**激活码**作为账号登录
- 无需密码（密码字段可任意填写）
- 首次登录时自动创建会员账号
- 适合软件授权场景

#### 用户模式 (Type=1)
- 用户需要**注册账号**后登录
- 需要用户名 + 密码
- 可配合激活码充值时间/点数
- 适合会员制软件场景

### 1.3 运营模式详解

| 模式 | 值 | 说明 |
|------|-----|------|
| 收费运营 | 0 | 正常运营，需要激活码 |
| 停止运营 | 1 | 暂停服务，所有 API 返回"已停止运营" |
| 免费运营 | 2 | 免费使用，无需激活码，自动分配时间 |

## 2. 登录规则配置

### 2.1 登录规则属性

```
登录规则 (ProjectLogin)
├── 基础设置
│   ├── Title: 规则名称
│   └── Mode: 登录模式 (绑定登录/普通登录/点数登录)
├── 注册设置
│   ├── RegMode: 注册模式 (带卡注册/普通注册)
│   └── EmailReg: 是否开启邮箱注册
├── 解绑设置
│   ├── UnbindMode: 解绑模式
│   ├── UnbindWeakenMode: 解绑扣时模式
│   ├── UnbindWeaken: 解绑扣时天数
│   ├── UnbindWeakenPoints: 解绑扣点数
│   ├── UnbindTimes: 最大解绑次数
│   ├── UnbindDate: 解绑周期 (天/月)
│   └── UnbindBefore: 解绑前置时间
└── 多开设置
    ├── NumberMode: 点数登录控制模式 (机器码/IP)
    ├── NumberMore: 最大限制数
    ├── NumberWeaken: 扣除点数
    ├── NumberWeakenTime: 扣点周期
    ├── PcMore: 单机最大在线数
    └── PcCodeMore: 最大多开上限
```

### 2.2 登录模式 (Mode) 详解

#### 绑定登录 (Mode=0)
**特点**：用户首次登录时绑定机器码，更换设备需要解绑

| 解绑模式 | 值 | 说明 |
|---------|-----|------|
| 不允许解绑 | 0 | 绑定后无法更换设备 |
| 原机解绑 | 1 | 只能在原设备上解绑 |
| 自动解绑 | 2 | 登录新设备时自动解绑旧设备 |
| 任意解绑 | 3 | 任何设备都可以解绑 |

**适用场景**：
- 需要限制设备授权的软件
- 防止账号共享
- 单设备授权许可

#### 普通登录 (Mode=1)
**特点**：不绑定机器码，可任意设备登录

**多开控制**：
- `PcMore=0, PcCodeMore=0`: 不允许多开，登录时踢掉其他设备
- `PcMore>0`: 单机最多同时在线数
- `PcCodeMore>0`: 同账号最多多开数

**适用场景**：
- 网页应用
- 允许多设备登录的软件
- 会员订阅服务

#### 点数登录 (Mode=2)
**特点**：每次登录/使用扣除点数，适合按次计费

| 参数 | 说明 |
|------|------|
| NumberMode | 控制模式：0=机器码控制, 1=IP控制 |
| NumberMore | 单用户最大可分发数量 |
| NumberWeaken | 每次扣除点数 |
| NumberWeakenTime | 扣点周期：0=每小时, 1=每天 |

**适用场景**：
- 按次计费服务
- API 调用计费
- 游戏点数系统

### 2.3 注册模式 (RegMode) 详解

#### 带卡注册 (RegMode=0)
- 注册时**必须**提供激活码
- 激活码的天数/点数自动充入账号
- 适合付费软件

#### 普通注册 (RegMode=1)
- 可免费注册账号
- 注册后需要激活码充值才能使用
- 适合试用后付费模式

## 3. 版本号管理

### 3.1 版本号属性

| 属性 | 类型 | 说明 |
|------|------|------|
| Version | float64 | 版本号（如 1.00, 2.50） |
| IsMustUpdate | int | 是否强制更新 (0=否, 1=是) |
| IsActive | int | 是否启用 (0=启用, 1=禁用) |
| Notice | string | 更新公告 |
| WgtUrl | string | 热更新文件下载地址 |

### 3.2 版本号格式

版本号在数据库中存储为 `float64`，显示时自动格式化：
- `1` → `1.00`
- `1.5` → `1.50`
- `2.00` → `2.00`

**API 请求时需要使用格式化后的版本号**，如 `1.00` 而非 `1`。

### 3.3 版本控制逻辑

```
客户端请求
    │
    ▼
检查版本号是否存在 ──否──▶ 返回错误：请创建对应可用的版本号
    │
    是
    │
    ▼
检查版本是否启用 (IsActive)
    │
    ├─ 启用 (0) ──▶ 正常使用
    │
    └─ 禁用 (1) ──▶ 返回错误：当前版本已禁用，请更新
           │
           ▼
       检查是否强制更新 (IsMustUpdate)
           │
           ├─ 是 (1) ──▶ 强制用户更新
           │
           └─ 否 (0) ──▶ 提示有新版本，可选更新
```

### 3.4 版本号使用建议

| 场景 | 版本号策略 |
|------|-----------|
| 内测阶段 | 创建低版本号，设置 IsActive=1 禁用 |
| 正式发布 | 创建正式版本，IsActive=0 启用 |
| 热更新 | 保持版本号不变，更新 WgtUrl |
| 大版本更新 | 新建版本号，旧版本设置 IsMustUpdate=1 |

## 4. 激活码管理

### 4.1 激活码类型 (Cards)

激活码类型定义了激活码的属性：

| 属性 | 类型 | 说明 |
|------|------|------|
| Title | string | 激活码类型名称（如"月卡"、"年卡"） |
| Price | float64 | 定价 |
| KeyPrefix | string | 4位前缀（如 "VIP1"） |
| Days | float64 | 充值天数 |
| Points | int | 充值点数 |
| Tag | string | 用户标签（如"VIP会员"） |
| KeyExtAttr | string | 扩展属性（自定义数据） |
| IsLock | int | 是否锁定 (0=正常, 1=锁定) |

### 4.2 激活码生成格式

激活码 = **前缀** + **随机码**

| 格式类型 | 值 | 示例 |
|---------|-----|------|
| 纯数字 | 0 | `VIP112345678901` |
| 大写字母数字 | 1 | `VIP1ABCD1234EFGH` |
| 小写字母数字 | 2 | `vip1abcd1234efgh` |
| 随机混合 | 3 | `VIP1aB3cD5eF7gH` |

### 4.3 激活码状态

| 状态 | UseTime | 说明 |
|------|---------|------|
| 未使用 | 0 | 可用于注册或充值 |
| 已使用 | >0 | 已绑定用户，不可再用 |
| 锁定 | IsLock=1 | 管理员锁定，暂停使用 |

### 4.4 激活码用途

#### 用途一：单码模式登录
```
用户输入激活码 → 直接登录 → 自动创建会员账号
```

#### 用途二：账号模式注册
```
用户注册账号 → 提供激活码 → 账号获得天数/点数
```

#### 用途三：账号充值
```
已有账号 → 使用激活码充值 → 增加天数/点数
```

## 5. 组合使用场景

### 场景一：单机授权软件

**配置**：
- 登录模式：单码
- 登录规则：绑定登录 + 自动解绑
- 激活码类型：天卡/月卡/年卡

**流程**：
```
1. 用户购买激活码
2. 首次登录：输入激活码 → 自动创建账号 → 绑定机器码
3. 后续登录：输入激活码 → 验证机器码 → 登录成功
4. 更换设备：自动解绑旧设备 → 绑定新设备
```

**SDK 示例**：
```go
client, _ := sdk.NewClient(sdk.Config{
    BaseURL:   "https://api.example.com",
    AppKey:    "your_appkey",
    SecretKey: "your_secretkey",
    Version:   "1.00",
})

// 单码模式：激活码作为用户名，密码任意
userInfo, err := client.Login(sdk.LoginRequest{
    Username: "798605218247", // 激活码
    Password: "any",          // 任意密码
})
```

---

### 场景二：会员订阅服务

**配置**：
- 登录模式：用户
- 登录规则：普通登录 + 带卡注册
- 激活码类型：月卡/季卡/年卡

**流程**：
```
1. 用户注册账号（需要激活码）
2. 登录使用服务
3. 到期后购买激活码充值续费
```

**SDK 示例**：
```go
// 注册（需要激活码）
err := client.Register(sdk.RegisterRequest{
    Username: "user@example.com",
    Password: "password123",
    Code:     "VIP12345678",  // 激活码

// 登录
userInfo, err := client.Login(sdk.LoginRequest{
    Username: "user@example.com",
    Password: "password123",
})

// 充值续费
err := client.Recharge(sdk.RechargeRequest{
    Username: "user@example.com",
    Key:      "VIP87654321",  // 新激活码
})
```

## 6. SDK 使用示例

### 6.1 完整工作流

```go
package main

import (
    "fmt"
    "time"
    
    "github.com/kerwin/network_valid/sdk/golang"
)

func main() {
    // 1. 创建客户端
    client, err := sdk.NewClient(sdk.Config{
        BaseURL:   "https://app.l9.lc",
        AppKey:    "your_appkey",
        SecretKey: "your_secretkey",
        Version:   "1.00",  // 注意格式为 X.XX
    })
    if err != nil {
        panic(err)
    }
    
    // 2. 登录
    userInfo, err := client.Login(sdk.LoginRequest{
        Username: "your_username",  // 或激活码
        Password: "your_password",
    })
    if err != nil {
        panic(err)
    }
    fmt.Printf("登录成功! Client Token: %s\n", userInfo.Client)
    fmt.Printf("到期时间: %s\n", userInfo.Endtime)
    
    // 3. 启动心跳（建议每30秒）
    go func() {
        ticker := time.NewTicker(30 * time.Second)
        defer ticker.Stop()
        for range ticker.C {
            if err := client.Heartbeat(); err != nil {
                fmt.Printf("心跳失败: %v\n", err)
                return
            }
        }
    }()
    
    // 4. 业务逻辑...
    
    // 5. 退出时下线
    defer client.Logout()
}
```

### 6.2 心跳机制说明

| 参数 | 值 | 说明 |
|------|-----|------|
| 心跳间隔 | 30秒 | SDK 建议值 |
| 缓存有效期 | 30分钟 | 后端 `h-{client}` 缓存 |
| 在线判断窗口 | 30分钟 | 后端查询最近30分钟的心跳 |

**心跳流程**：
```
登录成功
    │
    ▼
返回 client token
    │
    ▼
定时发送心跳 (每30秒)
    │
    ├─ 成功 ──▶ 更新缓存时间戳
    │
    └─ 失败 ──▶ 可能被踢下线或 token 过期
           │
           ▼
       需要重新登录
```

### 6.3 错误处理

```go
userInfo, err := client.Login(req)
if err != nil {
    if apiErr, ok := err.(*sdk.APIError); ok {
        switch apiErr.Errno {
        case 400:
            fmt.Println("用户不存在或密码错误")
        case 10009:
            fmt.Println("版本号不存在")
        case 10011:
            fmt.Println("未绑定登录规则")
        case 10006:
            fmt.Println("项目已停止运营")
        default:
            fmt.Printf("错误: %s (码: %d)\n", apiErr.Errmsg, apiErr.Errno)
        }
    }
    return
}
```

## 附录：常见问题

### Q1: 版本号格式错误
**问题**：请求返回"version不合法"
**解决**：使用格式化版本号，如 `1.00` 而非 `1` 或 `1.0`

### Q2: 心跳失败
**问题**：心跳返回"已下线"
**解决**：
1. 确认登录成功后保存了 client token
2. 确认心跳参数使用 `client` 而非 `username`
3. 检查是否被管理员强制下线

### Q3: 注册返回"已停止运营"
**问题**：项目状态正常但注册失败
**解决**：这是后端 bug（已修复），检查 `func.go:101` 是否使用 `StatusType` 而非 `Type`

### Q4: 激活码无效
**问题**：激活码无法使用
**解决**：
1. 检查激活码是否已使用
2. 检查激活码是否被锁定
3. 检查激活码归属项目是否匹配

## 7. 配额系统

### 7.1 概述

配额系统允许为套餐类型配置多维度使用限制，支持：
- **多维度配额**：如下载次数、流量、API调用次数等
- **多周期支持**：每日/每周/每月重置
- **永久套餐**：`days=0` 表示永久有效
- **免费套餐保底**：用户始终有免费套餐作为保底

### 7.2 套餐类型扩展字段

| 属性 | 类型 | 说明 |
|------|------|------|
| quota_rules | JSON | 配额规则（可选，NULL表示无限制） |
| is_free_tier | int | 是否为免费套餐 (0=否, 1=是) |
| priority | int | 套餐优先级（越大越高） |
| days | float64 | 天数，0表示永久 |

### 7.3 配额规则 JSON 格式

```json
{
  "quotas": [
    {
      "key": "download",       // 配额标识
      "name": "下载次数",       // 显示名称
      "limit": 20,             // 限额数量
      "period": "daily",       // 周期：daily/weekly/monthly
      "reset_day": 1,          // 重置日：daily=每日，weekly=周几(1-7)，monthly=几号(1-31)
      "unit": "count"          // 单位：count/bytes/custom
    },
    {
      "key": "traffic",
      "name": "流量",
      "limit": 1073741824,     // 1GB (单位：字节)
      "period": "monthly",
      "reset_day": 1,
      "unit": "bytes"
    }
  ]
}
```

### 7.4 套餐升降级规则

| 场景 | 行为 |
|------|------|
| 兑换高优先级套餐 | 立即升级，原套餐失效 |
| 兑换同优先级套餐 | 时长叠加 |
| 兑换低优先级套餐 | 进入排队，当前套餐到期后激活 |
| 付费套餐到期 | 自动降级为免费套餐 |

### 7.5 SDK 配额接口

```go
// 查询配额
quotas, err := client.CheckQuota([]string{"download", "traffic"})
for _, q := range quotas {
    fmt.Printf("%s: %d/%d (剩余 %d)\n", q.Name, q.Used, q.Limit, q.Remaining)
}

// 扣减配额
result, err := client.DeductQuota("download", 1)
if result.Success {
    fmt.Printf("扣减成功，剩余: %d\n", result.Remaining)
}

// 批量扣减
results, err := client.DeductQuotaBatch([]sdk.QuotaDeductItem{
    {Key: "download", Amount: 1},
    {Key: "traffic", Amount: 1048576}, // 1MB
})

// 获取当前套餐
plan, err := client.GetCurrentPlan()
fmt.Printf("当前套餐: %s, 状态: %s\n", plan.PlanName, plan.Status)
```

### 7.6 API 接口

| Action | 说明 |
|--------|------|
| quota.check | 查询配额（支持 keys 参数） |
| quota.deduct | 扣减单个配额 |
| quota.deductBatch | 批量扣减配额 |
| plan.list | 获取项目套餐列表 |
| plan.current | 获取用户当前套餐 |
| plan.queued | 获取用户排队套餐 |

### 7.7 预设套餐方案

项目创建时可选择预设套餐方案，默认提供：
- **免费套餐**：永久有效，优先级最低
- **标准套餐**：30天，可自定义

管理员可自定义预设方案的名称、时长、配额规则。

## 附录：常见问题（续）

### Q5: 配额扣减失败
**问题**：DeductQuota 返回"配额不足"
**解决**：
1. 先调用 CheckQuota 检查剩余配额
2. 确认套餐未过期
3. 确认配额周期已重置（每日/每周/每月）

### Q6: 套餐到期后无法使用
**问题**：套餐到期后服务不可用
**解决**：
1. 检查是否有免费套餐作为保底
2. 购买新的激活码充值
3. 确认定时任务正常执行（套餐到期检查）

### Q7: 概念变更说明
**说明**：系统进行了概念重命名
- 激活码类型 → 套餐类型
- 激活码 → 兑换码

API 和数据库字段保持不变，前端 UI 文案已更新。
